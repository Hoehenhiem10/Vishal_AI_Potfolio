Three Data Challenge Scenarios

1

Missing Data in Health Datasets

Problem: Age and weight values missing in healthcare dataset requiring robust imputation strategies for clinical applications.

Engineering Challenge: Healthcare data requires high accuracy and interpretability. Missing data patterns could indicate systematic issues that affect patient safety and treatment decisions.

2

Noisy Image Data in Autonomous Vehicles

Problem: Motion blur, low light conditions, and sensor noise degrading vision system performance in real-time autonomous vehicle applications.

Engineering Challenge: Real-time constraints demand efficient preprocessing while maintaining safety-critical object detection and navigation capabilities.

3

Imbalanced Data in Defect Detection

Problem: Only 5% defective products in manufacturing dataset creating severe class imbalance for quality control systems.

Engineering Challenge: False negatives in safety-critical manufacturing can lead to defective products reaching customers, requiring optimized precision-recall balance.

Scenario 1: Missing Data Solutions

Strategic Approach to Healthcare Data Imputation
1
Analyze Missingness Pattern

Determine if data is Missing Completely at Random (MCAR), Missing at Random (MAR), or Missing Not at Random (MNAR)

2
Quantify Missing Percentage

If <10% missing → Simple imputation (Mean/Median/KNN). If >10% → Advanced techniques required

3
Apply Robust Models

Use XGBoost which handles missing values natively without explicit imputation

4
Add Missingness Indicators

Create new binary features to capture patterns in missing data that may be clinically relevant

SMOTE vs ADASYN Comparison

SMOTE

Synthetic Minority Oversampling Technique

  • Creates synthetic examples using k-nearest neighbors
  • Generates samples along line segments between minorities
  • Uniform distribution of synthetic samples
  • Simpler implementation and faster processing
VS

ADASYN

Adaptive Synthetic Sampling

  • Extends SMOTE with adaptive density estimation
  • Focuses on minority samples harder to classify
  • Non-uniform sample generation based on difficulty
  • Better for complex decision boundaries
